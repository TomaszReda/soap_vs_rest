13. Serwer

Protokół SOAP jest ściśle powiązany z serwerem i ma z nim ścisłą umowę komunikacyjną, co utrudnia wprowadzanie zmian lub aktualizacji. Klient wchodzący w interakcję z interfejsem API REST nie potrzebuje wiedzy o interfejsie API, ale klient wchodzący w interakcję z interfejsem API SOAP potrzebuje wiedzy o wszystkim, czego będzie używał, zanim będzie mógł zainicjować interakcję.

Czyli tak po ludzku chodzi o to ze do SOAPA musi byc szczegółowa dokumentacja zanim czegoś użyjemy w zapytaniu
