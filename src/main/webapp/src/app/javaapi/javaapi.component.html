JAX-WS

Java API dla usług internetowych XML (JAX-WS) to API języka programowania Java do tworzenia usług internetowych, w szczególności usług SOAP. JAX-WS jest jednym z API programowania Java XML. Jest częścią platformy Java EE.

Specyfikacja JSR 224 JAX-WS 2.2 definiuje standardowe odwzorowanie Java na WSDL, które określa, w jaki sposób operacje WSDL są powiązane z metodami Java, gdy komunikat SOAP wywołuje operację WSDL. To odwzorowanie Java na WSDL określa, która metoda Java zostanie wywołana i jak ten komunikat SOAP jest odwzorowany na parametry metody.

To mapowanie określa również, w jaki sposób zwracana wartość metody jest mapowana na odpowiedź SOAP.

JAX-WS korzysta z adnotacji wprowadzonych w Javie SE 5, aby uprościć tworzenie i wdrażanie klientów i punktów końcowych usług WWW. Jest częścią pakietu Java Web Services Development Pack. JAX-WS może być używany w Javie SE od wersji 6. [1] JAX-WS 2.0 zastąpił JAX-RPC API w Java Platform, Enterprise Edition 5, który bardziej skupia się na usługach internetowych w stylu dokumentów.

Ten interfejs API stanowi rdzeń Project Metro w społeczności Open Source Application GlassFish firmy Oracle Corporation.

JAX-WS to także jedna z podstaw WSIT.

JAX-RS

JAX-RS, czyli Java API for RESTful Web Services jest specyfikacją zdefiniowaną w platformie Java EE od wersji 6. Obecna wersja 2.0 została wydana w roku 2013 i stanowi element Javy EE 7.


Założeniem JAX-RS było stworzenie API, które wymaga minimalnej konfiguracji, a sama konfiguracja odbywa się przy pomocy prostych adnotacji zastosowanych na poziomie odpowiednich klas i metod będących zwykłymi klasami POJO. W praktyce pozwala to w bardzo prosty sposób udostępnić w swojej aplikacji REST API.

Dostępne implementacje

JAXRS jest jedynie specyfikacją, zbiorem reguł, adnotacji oraz interfejsów, które powinny być zaimplementowane. Istnieje kilka bibliotek, które są implementację tego standardu. Referencyjna to biblioteka Jersey jeszcze z czasów firmy Sun. Korzystając z serwerów JBoss/Wildfly napotkamy na RestEASY, a oprócz tego istnieje jeszcze Apache CXF będąca jednocześnie implementacją specyfikacji JAX-WS (Java API for XML Web Services).
